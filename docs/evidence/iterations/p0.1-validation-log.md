# Priority 0.1 Batch Validation Log

## Purpose
This file is the single reporting ledger for 3-session batch submissions during simulator validation.

## Reporting Rules
- Use one dated entry per batch (`Quiet`, `Moderate`, `Noisy`).
- Keep matrix values in `device-test-matrix.md` as source-of-truth.
- Capture only new checklist coverage deltas in each batch entry.
- Record unresolved unknowns explicitly; empty means no unknowns remain.

## Milestone Commit Protocol
- **Milestone 1 (completed):** execution artifacts hardened.
- **Milestone 2:** after Quiet + Moderate batches are validated and recorded.
- **Milestone 3:** after Noisy batch is validated, unknowns are closed, and P0.1 is marked done.

Commit message templates:
- `docs(evidence): record P0.1 quiet and moderate batch validation`
- `docs(evidence): close P0.1 with noisy batch and zero unknowns`
